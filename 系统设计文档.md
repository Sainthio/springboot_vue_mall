3. 子系统模块设计 

3.1 管理端 

3.1.1	商品管理模块 
功能描述：支持管理员创建商品信息（含商品编号、名称、类型、品牌、规格、价格、库存）、设置秒杀倒计时结束时间、发布/关闭商品，基于富文本编辑器实现商品介绍排版。 
核心接口： 
POST /shangpinxinxi/save：新增商品信息 
PUT /shangpinxinxi/update：更新商品信息 
GET /shangpinxinxi/page：获取商品分页列表 
DELETE /shangpinxinxi/delete：删除商品信息 
GET /shangpinxinxi/info/{id}：获取商品详情 

3.1.2 商品类型管理模块 
功能描述：管理员管理商品分类信息，支持商品类型的增删改查。 
核心接口： 
POST /shangpinleixing/save：新增商品类型 
PUT /shangpinleixing/update：更新商品类型 
GET /shangpinleixing/page：获取商品类型列表 
DELETE /shangpinleixing/delete：删除商品类型 

3.1.3 订单管理模块 
功能描述：管理员查看所有订单信息，支持订单状态管理（已支付、已发货、已完成、已取消、已退款），支持按订单状态、时间进行数据统计。 
核心接口： 
GET /orders/page：获取订单分页列表 
PUT /orders/update：更新订单状态 
GET /orders/value/{xColumnName}/{yColumnName}：按值统计订单数据 
GET /orders/group/{columnName}：订单分组统计 

3.1.4 用户管理模块 
功能描述：管理员管理用户信息，查看用户列表、重置用户密码。 
核心接口： 
GET /yonghu/page：获取用户分页列表 
POST /yonghu/save：新增用户 
PUT /yonghu/update：更新用户信息 
DELETE /yonghu/delete：删除用户 
GET /users/page：获取管理员用户列表 

3.1.5 评论管理模块 
功能描述：管理员管理商品评论信息，支持评论内容的查看和回复。 
核心接口： 
GET /discussshangpinxinxi/page：获取评论分页列表 
PUT /discussshangpinxinxi/update：回复评论 
DELETE /discussshangpinxinxi/delete：删除评论 

3.1.6 活动管理模块 
功能描述：管理员发布购物活动信息，通过富文本编辑器编辑活动内容。 
核心接口： 
POST /news/save：新增活动信息 
PUT /news/update：更新活动信息 
GET /news/page：获取活动列表 
DELETE /news/delete：删除活动信息 

3.1.7 在线客服模块 
功能描述：管理员处理用户的客服咨询，支持问题查看和回复功能。 
核心接口： 
GET /chat/page：获取客服消息列表 
PUT /chat/update：回复客服消息 

3.2 用户端 

3.2.1 商品浏览模块 
功能描述：用户浏览商品列表、查看商品详情，支持按商品类型筛选、智能排序，查看商品点击统计。 
核心接口： 
GET /shangpinxinxi/list：获取商品列表（前端） 
GET /shangpinxinxi/detail/{id}：获取商品详情 
GET /shangpinxinxi/autoSort：智能排序商品 
POST /shangpinxinxi/thumbsup/{id}：商品点赞/点踩 

3.2.2 购物车模块 
功能描述：用户将商品加入购物车，管理购物车中的商品数量和价格。 
核心接口： 
POST /cart/add：添加商品到购物车 
GET /cart/list：获取购物车列表 
PUT /cart/update：更新购物车商品数量 
DELETE /cart/delete：删除购物车商品 

3.2.3 订单提交模块 
功能描述：用户通过购物车提交订单，选择收货地址，触发订单数据校验与防重提交机制，支持订单支付状态管理。 
核心接口： 
POST /orders/add：提交订单 
GET /orders/list：获取用户订单列表 
PUT /orders/update：更新订单信息 
GET /orders/detail/{id}：获取订单详情 

3.2.4 地址管理模块 
功能描述：用户管理收货地址，支持新增、编辑、删除地址，设置默认地址。 
核心接口： 
POST /address/add：新增收货地址 
GET /address/list：获取地址列表 
PUT /address/update：更新地址信息 
PUT /address/default：设置默认地址 
DELETE /address/delete：删除地址 

3.2.5 个人中心模块 
功能描述：用户查看历史订单记录、统计个人购买次数、管理个人信息、查看收藏列表。 
核心接口： 
GET /yonghu/detail/{id}：获取用户详情 
PUT /yonghu/update：更新个人信息 
GET /storeup/list：获取收藏列表 
POST /storeup/add：添加收藏 
DELETE /storeup/delete：删除收藏 

3.2.6 评论互动模块 
功能描述：用户对商品进行评论，查看其他用户的评论内容。 
核心接口： 
POST /discussshangpinxinxi/add：提交商品评论 
GET /discussshangpinxinxi/list：获取商品评论列表 

3.2.7 在线客服模块 
功能描述：用户向客服提交咨询问题，查看客服回复内容。 
核心接口： 
POST /chat/add：提交客服咨询 
GET /chat/list：获取客服消息列表 

4. 数据库设计 

4.1 系统逻辑结构设计 
采用 ER 图设计数据库逻辑结构，核心实体包括： 
商品信息（Shangpinxinxi）：存储商品基本信息（编号、名称、类型、品牌、规格、价格、库存、倒计时结束时间）。 
商品类型（Shangpinleixing）：存储商品分类信息（类型名称）。 
用户（Yonghu）：存储用户基本信息（账号、密码、昵称、性别、手机、邮箱、余额）。 
订单（Orders）：关联商品与用户，存储订单信息（订单编号、商品信息、购买数量、价格、状态、收货地址）。 
购物车（Cart）：关联商品与用户，存储购物车信息（商品ID、购买数量、单价）。 
地址（Address）：关联用户，存储收货地址信息（地址、收货人、电话、是否默认）。 
收藏（Storeup）：关联商品与用户，存储收藏信息（收藏类型、推荐类型）。 
商品评论（Discussshangpinxinxi）：关联商品与用户，存储评论信息（评论内容、回复内容）。 
购物活动（News）：存储活动信息（标题、简介、图片、内容）。 
在线客服（Chat）：关联用户与管理员，存储客服消息（提问、回复、是否回复）。 
管理员用户（User）：存储管理员信息（用户名、密码、角色）。 
Token（Token）：存储用户登录令牌信息（用户ID、用户名、角色、过期时间）。
