## 🏆 最推荐：23-极简纵向版

### 特点
- ✅ **宽度最窄**（适合A4纸）
- ✅ 纵向排列，从上到下
- ✅ 只显示核心实体（用户、订单、商品、类型）
- ✅ 属性精简，不冗余
- ✅ 完美适合打印和投影

### 包含内容
```
用户
  ├─ id
  ├─ 账号
  └─ 姓名
  ↓
创建订单
  ↓
订单
  ├─ id
  ├─ 订单号
  ├─ 状态
  └─ 总价
  ↓
包含
  ↓
商品
  ├─ id
  ├─ 名称
  ├─ 价格
  └─ 库存
  ↓
分类
  ↓
商品类型
  ├─ id
  └─ 类型
```

## 🎯 推荐：21-超窄版

### 特点
- ✅ **宽度很窄**
- ✅ 包含购物车（但不显示）
- ✅ 4个核心实体纵向排列
- ✅ 标注了秒杀关键字段（★）
- ✅ 适合Word文档和报告

### 包含内容
```
用户 → 创建订单 → 订单 → 包含商品 → 商品 → 分类 → 商品类型

关键属性：
- 商品.库存★（秒杀核心）
- 商品.秒杀时间★（秒杀核心）
```

## 🎤 答辩专用：22-答辩专用版

### 特点
- ✅ **中英文双语**
- ✅ 实体名标注英文（User、Orders、Product）
- ✅ 关系标注基数（1:n、n:1）
- ✅ 关键字段用金色标注
- ✅ 包含购物车分支

### 适合场景
- 答辩时讲解业务逻辑
- 需要展示系统架构
- 导师要求看到英文名

### 包含内容
```
用户 (User)
  ↓ 1:n
订单 (Orders)
  ↓ n:1
商品 (Product)
  - 库存★
  - 秒杀时间★
  ↓ n:1
商品类型 (Category)

+ 购物车分支 (Cart)
```

## 📐 详细版：20-纵向紧凑版

### 特点

- ✅ 所有关系都显示

## 🎤 答辩讲解建议

### 使用纵向E-R图讲解（2分钟）

**Step 1: 指着顶部（30秒）**
```
"这是用户实体，位于最上方，存储用户的基本信息，
包括账号、姓名等。"
```

**Step 2: 指着中间（1分钟）**
```
"用户通过'创建订单'关系与订单实体关联，是一对多关系。

订单实体记录交易信息，包含订单号、总价、状态等。

订单通过'包含商品'关系与商品实体关联，一个订单可以
包含一个商品。"
```

**Step 3: 指着底部（30秒）**
```
"商品实体是秒杀的核心，包含库存和秒杀结束时间两个
关键字段。

商品按类型分类，通过'分类'关系与商品类型实体关联。"
```
